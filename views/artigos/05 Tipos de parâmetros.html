05. Tipos de parâmetros

Query Params: Usado para filtros e paginação | req.query

Geralmente é utilizado dentro de uma rota que possui uma interrogação, um recurso utlizando query ficaria assim:

http://localhost:3000/projects?title=React&name=fulano

Como pegar esses valores no back-end?

server.get('/projects',(req, res) => {
    const { title, name } = req.query;

    res.json({ list: [title, name] })
});

Route Params || req.params

Usado para identificar recursos na hora de atualiza ou deletar:

http://localhost:3000/projects/:id

Como pegar esses valores no back-end?

server.put('/projects/:id', (req, res) => {
    const id = req.params;

    console.log(id);
    res.json({ id })
});

Request Params || req.body

Usado para criar algum dado do front para o back, quando se quer mandar um formulário para o backend o request params é muito utilizado.

Como pegar esses valores no back-end?

// E necessario configurar o epress para que receba o formato .json
server.user(express.json()); 

server.post('/projects', (req, res) => {
    const body = req.body;

    console.log(body);
    res.json({ body });
);
